## 1. 系统概述

**门店私域助手** 是一套基于微信小程序的门店营销与核销系统。旨在帮助门店解决以下问题：
*   **企微引流**：将企业微信会员转化为小程序数字化会员。
*   **精准营销**：通过手机号匹配，实现 1v1 发券和群发活动。
*   **高效核销**：员工端快速扫码核销优惠券，杜绝作弊。
*   **数据洞察**：实时监控发券、核销、营收数据。

## 3. 顾客端操作指南 (C 端)

### 3.1 授权入会
1.  扫描门店桌码或活动码进入小程序。
2.  点击弹窗中的 **「立即授权领取」**。
3.  允许获取手机号，系统自动同步会员身份。

### 3.2 查看/购买优惠券
1.  点击首页 **「购买优惠券」**。
2.  浏览可购买的代金券（如：100 元代金券）。
3.  点击 **「立即购买」** 或 **「免费领取」**。
4.  支付成功后，券将自动存入 **「我的优惠券」**。

### 3.3 出示核销码
1.  点击首页 **「我的优惠券」**。
2.  点击需要使用的券，进入详情页。
3.  向店员出示页面中的 **二维码**。

## 4. 员工端操作指南 (核销)

### 4.1 身份登录
*   员工首次使用需在后台 `staff` 表中录入信息（OpenID、姓名、角色）。
*   登录小程序后，系统自动识别员工身份，显示 **「员工核销」** 入口。

### 4.2 扫码核销
1.  点击首页/工作台中的 **「员工核销」**。
2.  点击 **「扫一扫核销」** 按钮。
3.  扫描顾客出示的二维码。
4.  **核对信息**：屏幕显示券名称、面值、规则。
5.  确认无误后，系统提示 **「核销成功」**。

### 4.3 查看到店顾客
*   在核销页面顶部，系统会显示 **「当前到店顾客」** 列表。
*   包含顾客姓名、会员等级（VIP/新客）、历史消费次数、偏好菜品等，方便员工提供个性化服务。

## 5. 管理端操作指南 (运营/店长)

### 5.1 券模板管理
*路径：首页 -> 总部 -> 券模板管理*

**功能**：创建和配置优惠券。
1.  **新增**：点击右上角 **「+ 新增」**。
2.  **填写信息**：
    *   **名称**：如“春季新品 8 折券”。
    *   **类型**：代金券（减钱）或 折扣券（打折）。
    *   **面值/折扣**：输入金额或折扣数。
    *   **售价**：顾客购买需支付的金额（填 0 为免费）。
    *   **库存**：填 `-1` 为无限库存。
    *   **有效期**：领券后的有效天数。
3.  **保存**：点击保存后，券模板即刻生效。

### 5.2 客户管理与手动发券 (核心功能)
*路径：首页 -> 总部 -> 客户管理*

**功能**：查看会员列表，并给指定会员发券。

**操作步骤**：
1.  **搜索客户**：在顶部搜索框输入客户手机号后 4 位或全号。
2.  **查看详情**：列表中显示客户的订单数、消费总额、持券数。
3.  **手动发券**：
    *   找到目标客户，点击下方的 **「🎁 赠送优惠券」** 按钮。
    *   在弹窗中确认发送（*注：当前默认发送预设模板，后续可升级为选择模板*）。
    *   提示 **「发送成功」** 后，客户的微信账户将立即收到该券。

### 5.3 营销活动与活动码生成 (核心功能)
*路径：首页 -> 总部 -> 营销管理*

**功能**：生成带参数的活动小程序码，用于企微群发。

**操作步骤**：
1.  滑动到页面底部，找到 **「📢 企微活动码生成器」**。
2.  **输入活动标识 (Scene)**：
    *   建议格式：`活动名_日期`，例如 `spring_sale_0403`。
    *   *注意：标识需唯一，用于后续统计不同活动的效果。*
3.  **生成**：点击 **「生成活动码」**。
4.  **保存**：长按生成的二维码图片，保存到手机相册。
5.  **发布**：将图片发送到企业微信群或私发给客户。

### 5.4 数据看板
*路径：首页 -> 总部 -> 数据看板*

**指标解读**：
*   **今日发券**：今天一共发出去多少张券（含购买和赠送）。
*   **今日核销**：今天实际使用了多少张券。
*   **今日收入**：通过售卖优惠券带来的直接收入。
*   **ROI (投入产出比)**：各营销规则带来的转化效果分析。

## 6. 核心业务场景 SOP

### 场景一：企微群发营销活动 (针对 1500 会员)

**目标**：唤醒沉睡客户，促进到店消费。

1.  **准备券**：在「券模板管理」中创建一张“老客专属 50 元券”，库存设为 1500，售价 0 元。
2.  **生成码**：在「营销管理」生成活动码，标识填 `old_customer_recall`。
3.  **群发**：
    *   打开企业微信 -> 客户联系 -> 群发助手。
    *   选择全部客户（或特定标签客户）。
    *   文案：“🎁 老会员福利！凭此码到店免费领 50 元代金券，点击长按识别 👇"
    *   附上刚才保存的二维码图片。
4.  **核销**：客户到店出示券，员工扫码核销。

### 场景二：1v1 精准发券 (针对高价值客户)

**目标**：挽回即将流失的 VIP 客户。

1.  **发现线索**：店长在企微看到某 VIP 客户抱怨“最近没去”。
2.  **查找客户**：打开小程序「客户管理」，搜索该客户手机号。
3.  **执行发券**：点击 **「🎁 赠送优惠券」**，系统自动发一张“挽留券”到客户账户。
4.  **通知客户**：在企微回复：“王总，刚特意给您申请了一张专属券，已经发到您微信卡包啦，期待您下次光临！”

## 7. 常见问题 (FAQ)

**Q1: 为什么提示“用户不存在”无法发券？**
*   **A**: 该客户虽然在你的企微里，但**还没有在小程序里授权过手机号**。
*   **解决**: 先发“活动码”引导他扫码授权，授权成功后再发券。

**Q2: 生成的活动码扫出来是首页，不是领券页？**
*   **A**: 活动码默认跳转首页。
*   **解决**: 首页会自动识别活动参数并弹出领券提示。如果未弹出，请检查代码中的场景值解析逻辑。

**Q3: 员工扫码提示“券已使用”？**
*   **A**: 该券已经被核销过，或者是假码。
*   **解决**: 请核对顾客手机上的二维码是否清晰，或检查核销记录。

**Q4: 如何查看某个活动的具体效果？**
*   **A**: 目前可通过数据库查询带有特定 `scene` 来源的券核销数量。后续版本将在看板中增加“活动效果分析”模块。
