## 1. 概述

「年年有喜」是面向餐饮门店的微信小程序，提供会员入会、优惠券购买与核销、营销自动化、顾客数据分析等功能。

**小程序名称：** 年年有喜
**入口方式：** 扫描门店桌贴二维码进入，或从微信搜索小程序直接打开

### 核心流程

```
客人扫码进入 → 授权手机号成为会员 → 跳转客如云点餐小程序 → 点餐消费
                                                    ↓
                                         支付后自动发放优惠券
                                                    ↓
                                         下次到店出示券码 → 员工扫码核销
```

### 支持门店

| 门店 | 门店 ID | 点餐小程序 |
|------|---------|-----------|
| 马己仙广东小馆 | 51866138 | 客如云（appId: wxdaa8741d326cf971） |
| 洪潮潮汕传统菜 | 64822111 | 客如云（appId: wx2f13889e1bd7b040） |

## 2. 角色说明

| 角色 | 说明 | 入口 |
|------|------|------|
| 客人（无角色） | 普通顾客，扫码入会、购买优惠券 | 小程序首页 |
| 员工（staff） | 门店服务人员，负责核销优惠券 | 首页 → 员工核销 |
| 管理员（manager） | 门店店长，可查看门店数据、撤销核销 | 首页 → 门店数据 + 员工核销 |
| 总部（admin） | 管理层，管理所有门店的券模板、营销规则、客户数据 | 首页 → 总部功能区 |

角色在云库 `staff` 集合中配置（openid + role + store_id + active），小程序启动时自动读取。

## 3. 客人端操作手册

### 3.1 扫码入会

**场景：** 客人首次到店，扫描桌上二维码

**操作步骤：**

1. 打开微信「扫一扫」，扫描桌上的小程序码
2. 自动进入小程序首页，显示「会员礼遇」页面
3. 页面展示门店名称、桌台编号（如有）
4. 点击 **「扫码点餐」** 按钮
5. 弹出手机号授权窗口，点击 **「允许」**
6. 系统自动完成：
   - 获取手机号，创建会员身份
   - 关联企业微信（如已配置）
   - 记录扫码到店日志
7. 授权成功后，按钮变为 **「去点餐」**，点击自动跳转至客如云点餐小程序

**注意：**
- 每个微信账号只需授权一次，后续到店自动识别
- 如为老会员，系统会自动同步历史积分
- 手机号信息安全保障，不会对外泄露

### 3.2 查看优惠券

**场景：** 客人已有优惠券，想查看或使用

**操作步骤：**

1. 进入小程序首页，点击底部菜单 **「我的优惠券」**（或首页入口）
2. 进入优惠券列表页，可看到三种状态的券：
   - **待使用**：未使用的券，显示券名称、面值、有效期
   - **已使用**：已核销的券
   - **已过期**：超过有效期的券
3. 点击任意一张券，进入详情页，显示：
   - 券名称与面值
   - 使用规则（如「全场通用，不找零」）
   - 有效期
   - 专属二维码（用于员工扫码核销）

### 3.3 购买优惠券

**场景：** 客人想在门店选购优惠券

**操作步骤：**

1. 进入小程序首页，点击 **「门店活动」** 或浏览可购买的券模板列表
2. 浏览可用优惠券模板，查看：
   - 券名称、类型（代金券 / 折扣券）
   - 面值与售价
   - 库存情况
   - 使用规则
3. 选择要购买的券，点击 **「购买」** 或 **「免费领取」**
4. **免费券：** 直接点击确认即可领取，即时到账
5. **付费券：** 跳转微信支付，完成付款后自动到账
6. 支付成功后，跳转至优惠券列表页查看已购买的券

### 3.4 使用优惠券（出示券码）

**场景：** 客人到店消费，向员工出示优惠券

**操作步骤：**

1. 进入小程序 → **「我的优惠券」** → 点击要使用的券
2. 进入详情页，页面显示专属二维码
3. 将手机屏幕展示给员工
4. 员工使用核销设备或手机扫码完成核销

**提示：**
- 每张券只能使用一次
- 部分券有最低消费门槛，结账前请确认金额满足条件
- 部分券有使用时段限制（如仅限用餐时段）

### 3.5 绑定企业微信

**场景：** 客人想通过企业微信接收优惠信息

**操作步骤：**

1. 进入小程序首页 → **「发送到企微」**（如有该入口）
2. 点击 **「绑定企业微信」**
3. 系统自动关联当前微信账号与企业微信
4. 绑定成功后，可将优惠券信息发送到企业微信

## 4. 员工端操作手册

### 4.1 进入核销页面

**前提：** 员工账号已由管理员在 `staff` 集合中录入（openid + role=staff + store_id + active=true）

**操作步骤：**

1. 打开小程序，自动识别员工身份
2. 首页出现 **「门店操作」** 区域
3. 点击 **「员工核销」**（图标：核）进入核销页面

### 4.2 扫码核销优惠券

**操作步骤：**

1. 进入核销页面，页面显示当前门店名称
2. **如有最低消费门槛的券**，在「本单消费金额」输入框中输入实际消费金额（单位：元）
3. 点击 **「扫码核销」** 按钮
4. 手机摄像头打开，对准客人出示的优惠券二维码
5. 扫码成功后，系统自动验证：
   - 券是否有效（未使用、未过期）
   - 券是否属于当前门店
   - 如有门槛，消费金额是否满足
   - 员工身份是否合法
6. **核销成功** → 页面显示绿色提示，自动刷新最近到店顾客列表
7. **核销失败** → 页面显示红色错误提示，常见原因：
   - 该券已使用
   - 该券已过期
   - 该券不可在本门店使用
   - 消费金额未达门槛
   - 操作过于频繁（3 秒内超过 8 次）

### 4.3 查看最近到店顾客

**操作步骤：**

1. 在核销页面下方，自动展示本门店最近 5 位到店顾客
2. 每位顾客显示：
   - 称呼（手机号后 4 位）
   - 等级标签（VIP / 常客 / 新客 / 未活跃 等）
   - 最近到店时间（今天 / 昨天 / 3 天前 等）
   - 来店次数
   - 最爱菜品

**提示：** 此信息帮助员工识别顾客身份，提供个性化服务。

### 4.4 核销注意事项

| 注意事项 | 说明 |
|---------|------|
| 门店匹配 | 只能核销当前门店的优惠券 |
| 重复核销 | 同一张券不可重复使用 |
| 金额门槛 | 部分券要求最低消费，需输入实际消费金额核验 |
| 时间限制 | 部分券仅在特定时段可用（如 10:00-22:00） |
| 星期限制 | 部分券仅在特定星期可用（如仅工作日） |
| 频率限制 | 3 秒内最多扫描 8 次，超出需等待 |

## 5. 管理员端操作手册（店长）

> 角色：manager，归属特定门店

管理员拥有员工端全部功能，并额外具备以下能力。

### 5.1 进入管理页面

**操作步骤：**

1. 打开小程序，自动识别管理员身份
2. 首页出现 **「门店操作」** 和 **「店长」** 区域
3. 点击 **「门店数据」**（图标：店）进入数据看板

### 5.2 查看门店数据看板

**功能：** 实时查看本门店当日经营数据

**页面信息：**

| 指标 | 说明 |
|------|------|
| 日期 | 当前统计日期（上海时区） |
| 今日发券数 | 当天自动发放 + 手动发放的优惠券总数 |
| 今日核销数 | 当天被员工核销的优惠券数量 |
| 今日收入 | 当天核销关联的消费金额（元） |
| 各规则 ROI | 每条营销规则的投入产出比 |

**操作：**

1. 下拉刷新，获取最新数据
2. 查看各营销规则的 ROI，判断规则效果

### 5.3 撤销核销

**场景：** 员工误操作，需要撤销已完成的核销

**操作步骤：**

1. 进入核销页面
2. 在核销记录中找到需要撤销的记录
3. 点击 **「撤销」** 按钮
4. 确认撤销

**限制条件：**
- 仅限 **10 分钟内** 完成的核销可撤销
- 仅限撤销 **本门店** 的核销记录
- 撤销后，优惠券恢复为「待使用」状态

## 6. 总部端操作手册

> 角色：admin，跨门店管理

总部端拥有全部功能权限，可管理所有门店的券模板、营销规则、客户数据，并查看全店汇总数据。

### 6.1 进入总部功能

**操作步骤：**

1. 打开小程序，自动识别总部身份
2. 首页出现 **「总部」** 区域，包含 4 个功能入口：

| 功能 | 图标 | 说明 |
|------|------|------|
| 券模板管理 | 券 | 创建、编辑、删除优惠券模板 |
| 营销管理 | 营 | 管理营销规则、生成活动二维码 |
| 客户管理 | 客 | 查看客户列表、手动发放优惠券 |
| 数据看板 | 数 | 查看全部门店汇总数据 |

### 6.2 券模板管理

**功能：** 创建和管理优惠券模板

#### 6.2.1 查看券模板列表

1. 点击 **「券模板管理」**
2. 页面展示所有券模板，每个模板显示：
   - 券名称
   - 类型（代金券 / 折扣券）
   - 面值（代金券显示金额，折扣券显示几折）
   - 售价（免费或具体金额）
   - 库存（-1 表示不限）
   - 是否启用（开关控制）

#### 6.2.2 创建新券模板

1. 点击右上角 **「新增」** 按钮
2. 填写表单：

| 字段 | 说明 | 示例 |
|------|------|------|
| 券名称 | 展示名称 | 「50元代金券」 |
| 类型 | 代金券(cash) / 折扣券(discount) | 代金券 |
| 面值 | 代金券：金额（元）；折扣券：折扣比例 | 50 |
| 售价 | 购买价格（元），0=免费 | 45 |
| 有效期 | 发放后有效天数 | 30 |
| 库存 | -1=不限，其他=具体数量 | 500 |
| 使用规则 | 文字说明 | 「全场通用，不找零」 |

3. 点击 **「保存」**

#### 6.2.3 编辑券模板

1. 在列表中点击要编辑的模板
2. 修改需要变更的字段
3. 点击 **「保存」**

#### 6.2.4 启用/停用券模板

- 在列表中，点击模板右侧的 **开关按钮**
- 开启 = 上架，客人可购买
- 关闭 = 下架，客人不可购买

#### 6.2.5 删除券模板

1. 在列表中点击模板的 **「删除」** 按钮
2. 弹出确认框，点击 **「确认」** 即可删除
3. **注意：** 删除后不可恢复

### 6.3 营销管理

**功能：** 管理营销自动化规则，生成活动二维码

#### 6.3.1 查看营销规则列表

1. 点击 **「营销管理」**
2. 页面展示所有营销规则，每条规则显示：
   - 规则名称
   - 触发条件（支付满 X 元 / X 天未到店等）
   - 发放的券模板
   - 是否启用
   - 当日 ROI（投入产出比）
   - 优先级

#### 6.3.2 启用/停用营销规则

- 点击规则右侧的 **开关按钮**
- 开启后，满足条件时自动发放优惠券
- 关闭后，该规则暂停执行

#### 6.3.3 调整规则优先级

- 点击规则旁的 **「+」** 或 **「-」** 按钮调整优先级
- 数字越大优先级越高
- 同一用户同一天，仅执行最高优先级且成功发券的一条规则

#### 6.3.4 生成活动二维码

**场景：** 创建营销活动码，用于企微群发、朋友圈推广

**操作步骤：**

1. 在营销管理页面，找到 **「活动码生成」** 区域
2. 输入活动标识（如 `spring_sale_0403`）
3. 点击 **「生成」**
4. 页面展示生成的小程序码图片
5. 长按保存图片，可用于印刷或线上分享
6. 扫码用户将自动进入小程序并带上活动标识参数

### 6.4 客户管理

**功能：** 查看客户列表，按手机号搜索，手动发放优惠券

#### 6.4.1 查看客户列表

1. 点击 **「客户管理」**
2. 页面顶部显示统计信息：
   - 总用户数
   - VIP 用户数
   - 新用户数
3. 下方展示客户列表，每位客户显示：
   - 手机号（部分隐藏）
   - 用户等级（VIP / 普通 / 新客）
   - 累计订单数
   - 累计消费金额
   - 优惠券数量
   - 注册日期
   - 最近到店时间

#### 6.4.2 搜索客户

1. 在顶部搜索框输入手机号
2. 点击搜索，列表实时筛选

#### 6.4.3 手动发放优惠券

**场景：** 向指定客户赠送优惠券（如补偿、生日礼等）

**操作步骤：**

1. 在客户列表中，找到目标客户
2. 点击该客户的 **「发券」** 按钮
3. 弹出可用券模板列表（仅显示当前门店可用且有库存的模板）
4. 选择要发放的模板
5. 确认发送
6. 系统自动将优惠券发放至该客户账户

**注意：**
- 仅当前门店有库存的模板可选
- 发放后，客户可在「我的优惠券」中查看

### 6.5 数据看板

**功能：** 查看全部门店汇总的经营数据

**操作步骤：**

1. 点击 **「数据看板」**
2. 页面显示 **「全部门店今日汇总」**，包含：
   - 今日总发券数
   - 今日总核销数
   - 今日总收入（元）
   - 各营销规则的 ROI
3. 下拉刷新获取最新数据

### 6.6 总部功能速查表

| 功能 | 路径 | 操作频率 |
|------|------|---------|
| 查看今日汇总数据 | 首页 → 数据看板 | 每日 |
| 创建券模板 | 首页 → 券模板管理 → 新增 | 按需 |
| 启用/停用券模板 | 首页 → 券模板管理 → 开关 | 按需 |
| 调整营销规则 | 首页 → 营销管理 → 优先级/开关 | 按需 |
| 生成活动码 | 首页 → 营销管理 → 活动码生成 | 营销活动时 |
| 手动发券 | 首页 → 客户管理 → 发券 | 按需 |
| 搜索客户 | 首页 → 客户管理 → 搜索 | 按需 |

## 7. 常见问题与故障排除

### 7.1 客人端

| 问题 | 原因 | 解决方法 |
|------|------|---------|
| 扫码后无法进入小程序 | 二维码过期或损坏 | 重新打印桌贴码 |
| 授权手机号失败 | 微信权限未开启 | 检查微信设置 → 隐私 → 授权管理 |
| 无法跳转点餐小程序 | 门店配置错误 | 联系管理员检查 config.js |
| 优惠券列表为空 | 尚未购买或领取 | 前往门店活动页购买 |
| 优惠券显示「已过期」 | 超过有效期 | 过期券不可恢复 |

### 7.2 员工端

| 问题 | 原因 | 解决方法 |
|------|------|---------|
| 无法进入核销页面 | 未录入 staff 集合 | 联系管理员添加员工记录 |
| 核销提示「无访问权限」 | openid 未匹配或 active=false | 联系管理员确认在职状态 |
| 核销提示「操作过于频繁」 | 3 秒内扫码超过 8 次 | 稍等 3 秒后重试 |
| 核销提示「该券不可在本门店使用」 | 券的门店与当前门店不匹配 | 确认客人是否在正确门店 |
| 核销提示「消费金额未达门槛」 | 输入金额低于券要求 | 确认实际消费金额 |

### 7.3 管理员/总部端

| 问题 | 原因 | 解决方法 |
|------|------|---------|
| 看板数据为空 | 当天无发券/核销记录 | 正常现象，有交易后自动出现 |
| 无法撤销核销 | 超过 10 分钟窗口 | 仅限 10 分钟内可撤销 |
| 券模板无法保存 | 必填字段未填写 | 检查券名称、面值、售价 |
| 手动发券失败 | 该模板在当前门店无库存 | 检查模板库存和门店配置 |

## 附录 E：核心业务场景 SOP

### 场景一：企微群发营销活动（针对 1500 会员）

**目标：** 唤醒沉睡客户，促进到店消费

1. **准备券**：在「券模板管理」中创建一张"老客专属 50 元券"，库存设为 1500，售价 0 元
2. **生成码**：在「营销管理」生成活动码，标识填 `old_customer_recall`
3. **群发**：
   - 打开企业微信 → 客户联系 → 群发助手
   - 选择全部客户（或特定标签客户）
   - 文案："老会员福利！凭此码到店免费领 50 元代金券，点击长按识别"
   - 附上刚才保存的二维码图片
4. **核销**：客户到店出示券，员工扫码核销

### 场景二：1v1 精准发券（针对高价值客户）

**目标：** 挽回即将流失的 VIP 客户

1. **发现线索**：店长在企微看到某 VIP 客户抱怨"最近没去"
2. **查找客户**：打开小程序「客户管理」，搜索该客户手机号
3. **执行发券**：点击「发券」，系统自动发一张"挽留券"到客户账户
4. **通知客户**：在企微回复："王总，刚特意给您申请了一张专属券，已经发到您微信卡包啦，期待您下次光临！"
