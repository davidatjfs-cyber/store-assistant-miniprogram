# 年年有喜会员小程序 — 操作手册

> 适用版本：微信小程序「年年有喜」 · 后台管理 HRMS (nnyx.cc)

---

## 一、角色权限说明

| 角色 | 能做什么 | 页面入口 |
|------|----------|----------|
| **顾客** | 扫码→授权→领券/买券→使用 | 小程序首页、券包、商城 |
| **店员 (staff)** | 核销客户优惠券、查看到店客户 | 小程序「员工核销」 |
| **店长 (manager)** | 店员权限 + 查看自己门店数据 | 小程序「门店数据」 |
| **管理员 (admin)** | 全部权限：券模板、营销规则、活动码、客户管理 | 小程序全部管理页 |

> 员工绑定：在云数据库 `staff` 集合中创建记录，字段包含 `openid`（员工微信号）、`role`（staff/manager/admin）、`store_id`（门店编号）、`active: true`。

## 二、券模板管理

### 2.1 入口
小程序 → 管理员首页 → 「券模板管理」 / 或 HRMS 对应界面

### 2.2 创建券模板
1. 点击「+ 新增」
2. 填写以下字段：

| 字段 | 说明 | 示例 |
|------|------|------|
| 券名称 | 顾客可见的券名 | 100元代金券 |
| 券类型 | **代金券**（固定面值）或 **折扣券**（打折） | 代金券 |
| 面值（元） | 代金券的面额 | 100 |
| 折扣 | 折扣券的折扣率 | 8.5（85折） |
| 售价（元） | `0` 表示免费领取，`>0` 表示需微信支付购买 | 0 或 68 |
| 有效期（天） | 自领取日起可用天数 | 30 |
| 库存 | `-1` 无限，其他数字为限制数量 | -1 或 500 |
| 使用规则 | 使用条件说明，显示在券详情页 | 全场通用，不找零，不兑现 |

3. 点击「保存」

### 2.3 管理券模板
- **启用/停用**：列表中的开关可控制券是否上架
- **编辑**：点击「编辑」修改已有模板
- **删除**：不可恢复，谨慎操作

### 2.4 顾客如何获得券
- **免费领取**：售价为 `0` 的券，顾客在商城直接点击「领取」
- **购买**：售价 `>0` 的券，顾客微信支付购买
- **自动发放**：营销规则（见第四章）根据用户行为自动发券
- **手动发放**：管理员在客户管理页面手动发券（见第五章）

## 三、活动二维码制作

### 3.1 用途
为每个线下营销活动生成专属小程序码，顾客扫码后进入小程序，系统会自动记录活动来源（`campaign_id`），实现：
- 区分不同渠道的获客效果
- 匹配新客归属到具体活动
- 追踪扫码→授权→领券→核销的完整漏斗

### 3.2 制作步骤
1. 进入小程序 → 管理员 → 「营销规则」
2. 在底部「企微活动码生成器」区域
3. 输入 **活动标识（Scene）**，如：
   - `马己仙_五一_001`
   - `洪潮_母亲节_2026`
   - `spring_campaign`
4. 点击「生成活动码」
5. 生成后长按图片保存，可通过企微群发给客户，或打印张贴到门店

> 生成的码是微信小程序码，扫码后打开「年年有喜」小程序首页，自动携带活动标识。

### 3.3 活动码命名规范建议
```
{门店缩写}_{活动主题}_{日期/序号}
```
例如：`MJX_新客立减_001`、`HC_下午茶_202605`

## 四、营销规则

### 4.1 概述
营销引擎会根据预设规则，在特定时机自动给用户发券。规则按**优先级**顺序匹配，每个用户每天只触发最高优先级的一条规则。

### 4.2 触发时机
| 时机 | 说明 |
|------|------|
| `post_authorization` | 用户授权手机号后（新会员欢迎） |
| `post_payment` | 用户支付购券后（消费后奖励） |
| `inactivity_scan` | 每日定时扫描沉睡用户（需配置 cron） |
| `manual` | 管理员手动触发 |

### 4.3 规则字段说明
每条规则包含：
- **规则名称**：自己识别用
- **优先级**：数字越小越优先，同级随机
- **触发时机**（hook 类型）
- **目标券模板 ID**：要发放的券
- **冷卻天数**：同一用户多久能再触发
- **每日用户上限**：每人每天最多触发几次
- **每日全局上限**：全店每天最多触发几次
- **7天内频次上限**：7天内最多触发几次
- **目标用户标签**：仅发给有特定标签的用户
- **自动禁用 ROI 阈值**：当 ROI 低于此值时自动停用
- **状态开关**：开启/关闭

### 4.4 管理规则
1. 进入「营销规则」页面
2. 可**开关**规则、调整优先级（+/- 按钮）
3. 查看每条规则的 ROI（投入产出比）

### 4.5 ROl 计算方式
```
ROI = 该规则发出的券产生的核销金额 / 券面值
```
ROI 低于阈值会自动停用。

## 五、客户管理与手动发券

### 5.1 客户列表
1. 进入「客户管理」页面
2. 查看所有注册用户，包括：手机号（脱敏显示）、会员等级、订单数、消费总额、券数量、最后到店时间
3. 可按手机号搜索

### 5.2 手动发券
1. 在客户列表中找到目标客户
2. 点击「发券」
3. 系统会用指定的模版发送一张券到客户账户
4. 客户打开小程序 → 券包即可看到

## 六、店员核销流程

### 6.1 前置条件
1. 店员已在 `staff` 集合中注册，`active: true`
2. 顾客已在小程序领券/买券

### 6.2 核销步骤
1. **顾客打开小程序** → 「我的券包」 → 点击要使用的券 → 显示**核销二维码**
2. **店员打开小程序** → 「员工核销」 → 点击「扫码核销」
3. 扫描顾客的券二维码
4. 系统自动校验：
   - 券是否未使用
   - 券是否在有效期内
   - 是否在本门店可用
   - 是否符合使用规则（最低消费等）
5. 核销成功 → 券变为「已使用」

### 6.3 可选操作
- **输入订单金额**：如有最低消费限制，需在扫码前输入订单金额（单位：分）
- **查看最近到店**：核销页面顶部显示最近到店客户信息（等级、偏好菜品、到店次数）

### 6.4 撤销核销（仅店长/管理员）
- 核销后10分钟内可撤销
- 撤销后券恢复为「未使用」

## 八、AI建议执行

### 8.1 概述
AI 系统根据用户画像、POS 消费数据、营销约束、历史案例等自动生成营销行动建议，辅助管理员决策。

### 8.2 使用流程
1. 进入 **AI建议执行** 选项卡
2. 点击「刷新建议」获取最新建议列表
3. 每条建议包含：
   - **建议标题**（如：向沉睡客户发放满减券）
   - **策略解释**：为什么选这个人群、为什么现在推、为什么用这个动作
   - **执行草稿**：标的门店、渠道、目标人群、文案摘要、预算
4. 针对每条建议可选择：

| 操作 | 说明 |
|------|------|
| **执行** | 确认执行，系统自动按草稿方案执行 |
| **修改后执行** | 调整参数后再执行 |
| **回填结果** | 执行后回填实际效果数据（核销数、营收等） |
| **忽略** | 标记为忽略，不再展示 |

### 8.3 建议状态
- **待处理**：新生成的建议，等待管理员处理
- **已执行**：已执行的建议
- **已忽略**：已忽略的建议

### 8.4 反馈机制
执行建议后，系统会自动弹出反馈表单，记录实际核销数和营收，用于持续优化 AI 策略。

## 九、POS 消费分析

### 9.1 概述
POS 消费分析展示来自客如云系统同步的到店消费数据，与小程序会员数据打通，帮助分析：
- 哪些菜品最受欢迎
- 什么时段客流最多
- 顾客的支付习惯
- 复购率和客单价

### 9.2 数据来源
每天凌晨 01:10 自动从飞书多维表格同步前一天 POS 数据。用户只需每晚将客如云导出的 Excel 粘贴到飞书表即可。

### 9.3 指标说明
| 指标 | 说明 |
|------|------|
| 总订单 | 所选时段内的 POS 总订单数 |
| 总收入 | 折后实际收入 |
| 客单价 | 平均每单消费金额 |
| 识别客户 | 有手机号可关联小程序会员的订单数 |
| 时段分布 | 各用餐高峰时段（10:00~21:00）的订单量和营收 |
| 支付方式 | 微信、支付宝、会员卡、现金等分布 |
| 菜品TOP15 | 按营收排序的热销菜品 |
| 门店对比 | 各门店的订单数、均消、总营收 |
| 复购分布 | 1次/2次/3次+消费的客户比例 |

### 9.4 时间筛选
支持近7天/近30天/近90天切换。

## 十、常见问题

### Q1: 顾客扫码后没反应？
- 确认小程序已发布正式版/体验版
- 确认二维码是本次活动码（通过「活动码生成器」制作）
- 检查顾客微信版本

### Q2: 核销失败怎么办？
- 检查券是否已使用或已过期
- 检查店员是否在 `staff` 表中且 `active: true`
- 检查券是否可在该门店使用
- 检查是否有最低消费限制

### Q3: 为什么POS消费没有数据？
- 确认飞书多维表格每晚已粘贴最新数据
- 同步在凌晨 01:10 自动执行，如需立即同步可联系管理员手动触发
- 检查飞书表列名是否与系统要求一致

### Q4: AI建议不准确？
- AI 建议基于历史数据和用户画像，数据越多建议越准
- 使用「回填结果」功能将实际效果反馈给系统，持续优化
- 可通过「营销约束」设置门店的折扣下限、禁用菜品等限制条件

### Q5: 如何添加新员工？
- 在云数据库 `staff` 表中新增记录
- 需要字段：`openid`（员工的微信openid）、`role`（staff/manager/admin）、`store_id`、`active: true`
- 员工用微信打开小程序后自动识别身份

### Q6: 活动码能用于非微信场景吗？
- 活动码是微信小程序码，只能在微信内扫码打开
- 如需在抖音/小红书推广，可使用海报工具生成图片版二维码，或在公域品宣模块管理
